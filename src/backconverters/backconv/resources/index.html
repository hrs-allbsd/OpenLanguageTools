Translation Technology:

At the moment I deal with translation memory technology.

The TMC Infrastructure/SunTrans project didn't work out that well (the external partner we were dealing with had severe engineering-process problems), and we have been left with a tool that works, but has many limitations. I'm writing translation memory tools, along with John Corrigan, that aim to fix most of the short-comings in SunTrans.

We have already written a comprehensive set of parsers, based around JavaCC which we use in the current software leveraging functionality of SunTrans. These are more detailed than simply calling the i18n message processing framework, per file-format, instead they parse the source message format itself - which is much more useful. Already, these parsers have found numerous bugs in Sun localisation files (eg. incorrectly formatted messages that the message compilers missed, duplicate messages, etc.)

I also wrote an SGML-sensitive sentence segmenter - one of the 1st things you need to be able to do in order to reuse translations from a previous product (cut a document up into sentences). I have a paper about this here. I'm not actively maintaining this, but we're planning on using a similar technique in the next version of SunTrans.

At the moment, I'm concentrating on trying to spec out the new version of SunTrans, and it's proving quite exciting. We're basing the whole thing on the SunONE software stack - and I already have a fair bit of code running on the Java 2 Enterprise Edition Reference Implementation 1.3 server. I'm currently working on getting this ported to SunONE Application Server 7 as well as writing other pars of the system, such as a HTML file filter and writing sentence segmenters.
