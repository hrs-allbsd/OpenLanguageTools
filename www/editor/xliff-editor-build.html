Introduction

This document aims to tell folks what they need to do to build the Open Language Tools XLIFF Translation Editor. It covers the build requirements and includes the output of a successful build so developers won't be put off by seeing occasional warning message produced during the build. We also have a similar document available explaining how to build the XLIFF Filters (which this document was based on).

As well as being interested in building the editor, readers may also be interested in reading the document About the Open Language Tools XLIFF Translation Editor, which provides a good introduction to the editor.

Build requirements

The Open Language Tools XLIFF Translation Editor doesn't have many build requirements. In order to build the editor, you need at least the Java 2 Standard Edition version 1.4.2 SDK. This can be downloaded here. The build also works using the Java 5 SDK - we haven't tested with other Java implementations : if you do build with other Java implementations please let us know, and we'll update this document.

We use Apache Ant to manage our build system, and require at least release version 1.6.2 - this and later versions of Ant can be downloaded here.

Other third party libraries and applications which are required during the build are bundled as part of the workspace and don't need to be downloaded separately. You can view these libraries, and the licences terms governing their use in the utilities/jars directory.

The source code can be checked out of the workspace by following the CVS setup guidelines - we strongly recommend developers use ssh to secure their CVS communication with our repository : more information on this can be found in the document Tunneling with SSH

Build command and output

To build the Open Language Tools XLIFF Translation Editor, simply issue the command :

% ant transeditor

The built binaries will be dropped in the newly created ./build directory at the top level of the workspace.

Note that some of the JavaCC Parsers can cause the Java compiler's virtual machine to exceed the boundary of it's stack - if you encounter an error message :

            The system is out of resources.
            Consult the following stack trace for details.
            java.lang.StackOverflowError

then you should set an environment variable ANT_OPTS to the value -Xss8192000 before running the ant command. Depending on which operating system or command shell you use, the way to set this variable will vary. On UNIX, running ksh, this setting is : 

                % ANT_OPTS=-Xss8192000
                % export ANT_OPTS

This problem has been fixed in later versions of the Java SDK, so setting this environment variable may not be required for all users.
